# AI 视频二创自动化工具：产品说明与技术落地方案

## 一、项目定位

### 项目名称（暂定）
AI 视频二创工厂

### 核心目标
输入一个视频链接（YouTube、TikTok 等），自动完成：

- 视频下载
- 字幕提取
- AI 翻译/改写
- AI 解说文案生成
- 自动配音
- 自动剪辑
- 自动字幕生成
- 导出新视频

最终输出：

一个具备“二次创作特征”的新视频文件。

---

# 二、核心用户

## 目标用户

### 1）AI 搬运工作室
需求：
- 批量处理视频
- 自动翻译
- 自动二创
- 降低人工成本

---

### 2）短视频创作者
需求：
- 快速生成解说视频
- 获取海外内容素材
- 自动生成中文内容

---

### 3）自媒体运营者
需求：
- 热点视频本地化
- 自动字幕
- 自动配音
- 快速产出内容

# 三、产品核心功能

# 功能模块一：视频链接解析

## 功能说明
支持输入：
- YouTube
- TikTok
- Bilibili
- X/Twitter
- Instagram

自动解析视频真实地址。

---

## 技术方案

### 核心工具
- yt-dlp

### 工作流

```text
用户输入 URL
↓
yt-dlp 获取真实流媒体地址
↓
下载音视频
↓
ffmpeg 合并
```

---

## Python 示例

```python
import yt_dlp

url = "https://youtube.com/..."

ydl_opts = {
    'format': 'bestvideo+bestaudio/best',
    'outtmpl': 'downloads/%(title)s.%(ext)s'
}

with yt_dlp.YoutubeDL(ydl_opts) as ydl:
    ydl.download([url])
```

---

# 功能模块二：音频提取

## 功能说明
从视频中提取人声音频。

---

## 技术方案

### 核心工具
- ffmpeg

### 命令

```bash
ffmpeg -i input.mp4 -vn -acodec mp3 output.mp3
```

# 功能模块三：语音识别（ASR）

## 功能说明
自动识别视频中的语音内容，并生成时间轴字幕。

---

## 推荐方案

### WhisperX

优点：

- 单词级对齐
- 支持多语言
- 精度高

---

## 工作流

```text
音频
↓
WhisperX
↓
JSON 字幕
```

---

## 输出结构

```json
[
  {
    "start": 1.2,
    "end": 3.5,
    "text": "hello everyone"
  }
]
```

# 功能模块四：AI 翻译与改写

## 功能说明
不是直接翻译。

而是：

将原字幕转化为：
- 解说风格
- 中文口语化
- 短视频节奏化
- 更具情绪表达

---

## 为什么不能直接翻译

直接翻译：
- 容易机械化
- 容易被平台识别重复
- 视频节奏差

必须加入：
- AI 理解
- AI 改写
- AI 解说化

---

## 推荐方案

### LLM 接口

第一阶段：
- DeepSeek
- OpenRouter

第二阶段：
- GPT-4.1
- Claude

---

## Prompt 示例

```text
你是短视频解说创作者。

请将以下英文字幕：

1. 改写成中文短视频解说风格
2. 更口语化
3. 增强情绪表达
4. 保持原视频核心信息
5. 每句长度适合配音
```

---

# 功能模块五：TTS 配音

## 功能说明
将 AI 改写后的内容自动生成中文配音。

---

## 第一阶段推荐

### Edge-TTS

优点：
- 免费
- 中文效果优秀
- 部署简单

---

## 第二阶段推荐

### GPT-SoVITS

或：

### CosyVoice

优点：
- 更自然
- 支持音色克隆
- 更接近真人

---

## 技术重点

每句话：
- 单独生成 TTS
- 单独控制时间轴

不能整段生成。

---

# 功能模块六：视频时间轴同步

## 核心难点

中文配音长度：
通常与原视频不一致。

例如：
- 英文 5 秒
- 中文可能只有 3 秒

会导致：
- 空镜头
- 对不上嘴型
- 节奏错乱

---

## 推荐解决方案

### 方案一：动态语速

根据时间轴：
自动调整 TTS 语速。

---

### 方案二：AI 控制文案长度

Prompt 中限制：

```text
保持与原字幕接近的时长
```

---

### 方案三：视频局部变速

使用 ffmpeg：

```bash
ffmpeg setpts
```

动态调整局部片段时长。

---

# 十、核心产品形态（重要）

# 本地 AI 视频工作流工具

本产品核心定位：

不是：

“全自动云端 SaaS 平台”

而是：

“本地 AI 视频二创工作流工具”。

---

# 核心特点

## 1）全部本地处理

视频：
- 本地下载
- 本地转码
- 本地 Whisper
- 本地 ffmpeg
- 本地导出

避免：
- GPU 服务器成本
- 视频存储成本
- 云端带宽成本

---

## 2）支持手动分析与字幕重构

系统：
不是纯自动生成。

而是：

允许用户：
- 手动修改字幕
- 手动改写解说
- 手动重组结构
- 手动调整节奏
- 手动删减片段
- 手动控制 Hook

---

## 推荐 UI 结构

```text
原字幕
↓
AI 改写建议
↓
用户手动编辑
↓
生成最终解说
```

---

# 为什么重要

因为：

纯自动 AI 二创：
- 很容易机械化
- 容易重复
- 原创度低

而：

“AI 辅助 + 人工重构”

更容易：
- 提升原创度
- 提升视频质量
- 提升平台通过率

---

## 3）支持用户自定义 API Key

系统默认：
不内置高成本 API。

用户可以自行填写：
- OpenAI API Key
- DeepSeek API Key
- OpenRouter API Key
- Gemini API Key

---

## 好处

### 对开发者

无需承担：
- 长期 API 成本
- 大规模调用费用
- 用户增长后的 Token 压力

---

### 对用户

可以：
- 自由切换模型
- 控制成本
- 控制质量
- 使用自己的账户额度

---

## 推荐系统结构

```text
设置页面
↓
填写 API Key
↓
本地加密保存
↓
本地调用模型接口
```

---

## 4）支持纯本地模式

即使不填写 API Key。

也可以：
- 视频下载

- 字幕编辑
- 本地导出
